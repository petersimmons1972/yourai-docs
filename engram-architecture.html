engram-go — System Architecture

yourai.com · internal technical reference · 2026-04-29

Overview

engram-go is the memory layer powering yourai.com. It stores, indexes, and retrieves memories for AI agents using PostgreSQL with pgvector for semantic search, Ollama for local embeddings, and a multi-tenant ownership model enforced at the database engine level via Row Level Security.

The system exposes two transports: an MCP (Model Context Protocol) server over SSE for AI clients such as Claude Code, and a REST API for web clients that cannot speak MCP natively. Both transports share the same service layer and the same database. Same memory, different wire format.

Single-user and multi-tenant in one binary. Existing single-user deployments require zero configuration changes. ENGRAM_API_KEY bootstraps a default owner on every startup — this is permanent infrastructure, not a migration shim.

Architectural constraint: REST handlers in internal/rest/ call internal/service/ only. No MCP-specific types may appear in the REST package. A future Rust gateway becomes a translation job (~200 lines), not a rewrite.

Multi-Tenant Owner Model

Identity

Every API request carries a Bearer token. The auth middleware computes SHA256(token), looks it up in api_keys, and places owner_id into the request context. An in-process LRU cache (TTL: ENGRAM_AUTH_CACHE_TTL, default 1 hour) avoids re-hashing on every call. Revocation calls cache.Delete immediately — it does not wait for TTL expiry.

Database isolation

The service layer issues SET LOCAL app.current_owner_id = $1 at the start of every transaction. RLS policies on all data tables use this session variable to filter rows. No query can return rows belonging to a different owner. PgBouncer in transaction mode is confirmed safe: SET LOCAL resets when the transaction ends, which is also when PgBouncer returns the connection to the pool.

Project scoping

Projects are free-form labels, unique per (owner_id, project) pair. Two owners can both use the project name "default" with no collision. Owner identity is resolved from the Bearer token — callers never prefix project names, and the API is unchanged for existing single-user deployments.

Key Schema — api_keys

Keys are never hard-deleted. Revocation sets revoked_at. The table is a permanent forensic record.

CREATE TABLE api_keys (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  owner_id      UUID NOT NULL REFERENCES owners(id),
  key_hash      TEXT NOT NULL UNIQUE,   -- SHA256(bearer_token)
  display_name  TEXT,
  revoked_at    TIMESTAMPTZ,            -- NULL = active
  last_used_at  TIMESTAMPTZ,            -- updated async on auth
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

Locked Architectural Decisions

D1 · Language
What language for new services?
Go core stays. New standalone services default to Rust. engram-go is mature Go. Rust for new boundary services for performance and safety.
D2 · Database placement
Where does the database live?
External to the app cluster always. App containers are ephemeral. DB state must survive container rebuilds and K8s pod restarts. TruNAS:5433 for homelab; Aurora for production.
D3 · Container images
Which container base images?
Chainguard hardened distroless images. Minimal CVE surface, no shell in production, supply chain provenance.
D4 · Data isolation
How is owner isolation enforced?
PostgreSQL RLS — not application-layer filters alone. Application-layer filters can be bypassed by bugs or missing middleware. RLS enforces isolation at the DB engine level regardless of how a query arrives.
D5 · Audit log
How is the audit log written?
Embedded, atomic with the operation it records. Audit entries are written in the same transaction as the operation. Append-only table.
D6 · Connection pooling
PgBouncer and SET LOCAL behavior?
PgBouncer transaction mode. SET LOCAL confirmed safe. Resets at transaction end = pool return point. RDS Proxy behavior with SET LOCAL is untested — POC required before Aurora production cutover.
D7 · Anthropic API keys
How do tenants supply Anthropic API keys?
BYOK + platform-absorbed. billing_model enum on the owners table. NULL anthropic_key_secret_ref implies claude_code subscription mode.

Tool Dispatch Timeout

All MCP tool handlers wrap execution in context.WithTimeout with a default of 15 seconds. Context propagation through mcp-go's in-process transport was validated via a spike test (PR #381): a blocking handler cancelled correctly within 300ms. Timed-out tools return IsError: true with a human-readable message rather than hanging the SSE connection.
